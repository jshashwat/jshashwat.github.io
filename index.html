Throughput Calculator for Queues: HoG vs YOLO

Kriti Kansal and Shashwat Jain
Fall 2021 ECE 4554/5554 Computer Vision: Course Project
Virginia Tech

Abstract and Introduction

Customer experience is one of the most important factors that define the success of any business. In a busy world like ours, nobody wants to wait in long queues. It can be just a coffee shop or a queue at the airport! As technology gets more advanced day by day, it is becoming easier to install computer vision techniques at places like these to help in reducing the waiting time for the customers. In this project, we will implement a computer vision model to calculate the throughput of queues in real-time. This will enable us to dynamically direct the customers to the queues that have better throughput and less waiting time, efficiently managing the intra-space traffic.

We will use two techniques to calculate the throughput of queues: HoG (Histogram of Oriented Gradients) and YOLO (You Only Look Once), a Deep Learning model for object detection. We want to compare the results of these two techniques for queue tracking applications. Although YOLO is expected to work very well with slow as well as fast paced objects in video feeds, we expect the results from HoG to be comparable because of relatively slow paced movement of objects in video feeds for queuing applications.

Training Pipeline

We will develop a user interface that takes in input feed from camera in real-time or recorded video feeds. The video frames are fetched and fed to our backend implementation where we process the frame independently using two object detection techniques 1) HOG (using our pre-trained HOG Detector) and 2) YOLO Object Detection. We locate an imaginary trip line at the start of the queue and monitor the location of our detected objects(people) with respect to this trip line. Whenever a person crosses the trip line, we calculate the throughput time and average it cumulatively. This tells us real-time efficiency of the queue which can be used as an information to monitor such data for multiple queues concurrently and manage the traffic accordingly.

Experiments and Results


Code Development and Usage: We will develop our own code for the HOG detector training and data set preprocessing. Preprocessing includes extracting objects(people) from the training dataset and feed them for training our HOG detector. Additionally the input video feed to our framework will be stabilized. We will also develop a graphical user interface to showcase realtime detection and throughput calculation. We will borrow YOLO as an existing technique and use our trained HOG detector for object detection.

Dataset Used: Collective Activity Dataset : We will leverage Class ID 4 in the dataset as our Data for Queuing Application. Other class IDs will provide a good dataset for detector training.

Project Success:

  • Succesfully training HoG on our training data set and collecting the throughput data.
  • Sucessfully executing YOLO on our data set and collecting the throughput data.
  • Compare and draw conclusions from the results.

    • Expectations and Potential Outcomes: Our expectation is for the HOG Object Detection to perform relatively closer to YOLO for the application use case. This is based on the fact that relative motion of objects in queues is very slow which is accompanied by near fixed scaling depth for objects for a fixed camera system at a certain orientation. The cumulative average throughput times for both the techniques and people counter will be deciding factors for accuracy comparisons between the two. If there are shortfalls in the HOG technique, we wish to investigate it. For future scope, a hybrid technique which leverages intermittent information from applying YOLO whenever the confidence relating to detection using HOG goes below a threshold can be deployed.
